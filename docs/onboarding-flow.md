# Web Onboarding Flow (会員登録 -> CLI初期設定)

## 目的

- 新規ユーザーが迷わず `初回同期` まで到達できること
- 通常機能の画面と初期設定の画面を分離し、認知負荷を下げること

## 体験ゴール

1. 会員登録
2. 組織の自動作成
3. クライアント1件作成
4. CLI接続（`npx invoiceai setup`）
5. ダッシュボード遷移

## ルーティング方針

- 初回導線: `/setup`
- 通常導線: `/org/:orgSlug/...`
- `/` は状態で振り分ける
  - クライアント 0 件: `/setup`
  - クライアント 1 件以上: `/org/:orgSlug`

## `/setup` の責務

- ステップ1: クライアント作成（最小入力）
- ステップ2: CLI接続案内（コマンド提示 + コピー）
- freee連携は必須にしない（後で設定）

## 実装ルール

- 初回ユーザーの組織はサーバー側で自動作成する
- 初期設定の完了判定は「クライアント1件以上」で行う
- 画面文言はユーザー向け日本語（「初期設定」）に統一する
- 通常画面のサイドバーや機能メニューは初期設定中に見せない

## 今後の拡張

- CLI接続完了の自動検知（API heartbeat）
- 初回同期完了をステップ3として表示
- `/setup` 進捗の永続化（再開導線）
