# InvoiceAI プロダクト戦略

## ペルソナ

### A. フリーランス / 個人事業主（1人）

自分の稼働時間を記録し、クライアントへの請求書を作成・送付したい。
GitHub の活動ログから稼働実績を自動で集計し、手入力の手間を減らしたい。

### B. 小規模受託開発会社の管理者

複数メンバーの稼働をクライアント別に管理し、請求書を一括生成したい。
メンバーの稼働状況を把握し、未入力のフォローや月次レポートを確認したい。

### C. 受託開発会社のスタッフ（メンバー）

日々の稼働時間を手早く入力し、タイムシートを確認したい。
できれば GitHub の活動から自動で入力されて、確認だけで済むのが理想。

### D. 業務委託エンジニアを複数抱えるクライアント企業

業務委託スタッフがタイムリーに請求書を発行してくれない問題を解決したい。
委託先の稼働状況をリアルタイムに把握し、請求プロセスを効率化したい。

## ペルソナ別 機能マトリクス

### A. フリーランス / 個人事業主

| 分類             | 既存機能                            | 不足機能                                        |
| ---------------- | ----------------------------------- | ----------------------------------------------- |
| 稼働管理         | 日別の稼働時間入力、タイムシートPDF | -                                               |
| 請求             | 請求書作成、freee連携、PDF出力      | -                                               |
| GitHub           | リポジトリ紐付け、活動から稼働提案  | -                                               |
| CLI              | login, setup, whoami                | 稼働入力（`cli log`）、稼働確認（`cli status`） |
| オンボーディング | setup フロー                        | -                                               |

概ね揃っている。CLI で稼働入力まで完結できると便利。

### B. 小規模受託開発会社の管理者

| 分類         | 既存機能                      | 不足機能                                     |
| ------------ | ----------------------------- | -------------------------------------------- |
| メンバー管理 | 招待、ロール管理              | -                                            |
| 稼働管理     | 全スタッフの月別稼働一覧      | メンバー別の稼働状況サマリー、未入力アラート |
| 請求         | クライアント別請求書、一括PDF | 月次一括請求書生成、請求ステータス管理       |
| GitHub       | App連携、ユーザーマッピング   | -                                            |
| クライアント | CRUD、freeeインポート         | -                                            |
| レポート     | なし                          | 月次売上サマリー、クライアント別収益レポート |

管理ダッシュボード系が薄い。「今月の全体像をパッと見たい」ニーズに応えられていない。

### C. 受託開発会社のスタッフ（メンバー）

| 分類     | 既存機能                  | 不足機能                               |
| -------- | ------------------------- | -------------------------------------- |
| 稼働入力 | Web UI で日別入力         | モバイル対応、入力リマインダー         |
| 確認     | タイムシートPDF           | 自分の月次サマリー（合計時間、残業等） |
| GitHub   | 活動から稼働提案          | 自動入力（提案の承認だけで済む）       |
| CLI      | setup（リポジトリ紐付け） | `cli log`（ターミナルから稼働報告）    |

入力の手間を減らす方向の機能が不足。「GitHub活動 → 自動で入力 → 確認だけ」が理想。

### D. 業務委託を発注するクライアント企業

| 分類     | 既存機能 | 不足機能                                           |
| -------- | -------- | -------------------------------------------------- |
| 稼働確認 | なし     | 委託先の稼働状況閲覧（読み取り専用ダッシュボード） |
| 請求受領 | なし     | 請求書の受領・承認フロー                           |
| レポート | なし     | 委託先別の月次稼働サマリー                         |
| 通知     | なし     | 請求書到着通知、稼働レポート定期送信               |

**現状ほぼ機能がない。** ここが一番のギャップ。
