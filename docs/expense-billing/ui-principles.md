# UI 設計原則: 月次経費請求

## 一般的なベストプラクティス

### 1. 視覚的階層（Visual Hierarchy）

**最も重要な情報が最も目立つ。**

- 数字（金額）はフォントサイズ・太さで階層をつける
  - 合計額 > 明細の金額 > 内訳の金額
- アクション（ボタン）は1画面に主ボタン1つ。副次的操作はテキストリンクかゴーストボタン
- 色は意味で使う: primary=主アクション、muted=補足情報、destructive=注意

**アンチパターン:** 全部同じ大きさで並べる。フォームと一覧が同じ重さで存在する

### 2. プログレッシブ・ディスクロージャー（Progressive Disclosure）

**必要になるまで見せない。**

- 初期状態は最小限。操作に応じて詳細が出る
- 例: 「内訳を追加」を押すまで内訳フォームは非表示
- 例: 為替レートは自動値を表示。「変更」を押したときだけ入力欄に切り替わる
- 設定項目が10個あっても、一度に見せるのは3個まで

**アンチパターン:** 全フィールドを最初から表示。使わない選択肢まで見せる

### 3. 直接操作（Direct Manipulation）

**データを見ているその場で編集できる。**

- 金額を変更したければ、金額の表示部分をクリックして編集
- 別画面に遷移して戻ってくる必要がない
- インライン編集 > モーダルダイアログ > ページ遷移

**アンチパターン:** 「編集」ボタン→別画面→フォーム入力→保存→戻る

### 4. フィードバック（Feedback）

**操作の結果が即座にわかる。**

- 保存したら即座にUIに反映（楽観的更新 or fetcher の自動 revalidation）
- 経費を追加したら、一覧に即追加される
- 請求書作成したら、結果（請求書番号、金額）を同じ画面で表示
- エラーはフィールドの隣に。ページ上部のまとめエラーだけだと対応しづらい

### 5. 一貫性（Consistency）

**同じパターンは同じ見た目・操作。**

- 「追加」は常に同じ位置（セクションの下部 or 右上）
- 「削除」は常に同じアイコン・位置（行の右端）
- 金額の表示形式を統一: `¥6,729` or `6,729円`、混在しない
- 通貨表示を統一: `$45.00` or `USD 45.00`、混在しない

### 6. エラー防止（Error Prevention）

**間違えにくい設計 > エラーメッセージ。**

- 通貨が USD のグループに JPY のアイテムは追加できないようにする（選択肢から消す）
- 金額フィールドに文字を入力できないようにする（input type 制約）
- 破壊的操作（削除）は確認ダイアログを挟む
- 為替レートの手動変更は「理由」の入力を求める（うっかり変更防止）

### 7. 空状態（Empty State）

**データがないときこそ、最も丁寧なガイダンスを。**

- 何を・なぜ・どうやって設定するかを説明
- 「追加」ボタンを目立たせる
- できれば具体例を見せる（「例: サーバ通信費 月$45」）
- 空状態は初回だけでなく、全削除後にも出る。同じ丁寧さで

### 8. 情報密度の最適化

**多すぎず少なすぎず。**

- 金額系UIは一覧性が重要。1行に収まる情報量にする
- テーブル > カードリスト（数値を比較しやすい）
- ただしモバイルではカードのほうが読みやすい場合もある
- 数値は右揃え + tabular-nums で桁を揃える

### 9. 導線の明確さ（Wayfinding）

**今どこにいて、どこに行けるかが常にわかる。**

- パンくずリスト or 「← 戻る」リンク
- 請求書作成 ⇄ 経費設定 の双方向リンク
- タブで現在の位置を示す
- 設定変更後に「請求書作成に戻る」リンクを表示

### 10. アクセシビリティ

- フォームには必ず `<Label>` を対応づける
- キーボードだけで操作完了できる
- 色だけで状態を伝えない（アイコンやテキストも併用）
- コントラスト比を確保

## オブジェクト指向 UI（OOUI）

### 核心: 名詞 → 動詞

ユーザーはまず**対象（オブジェクト）を選び**、次に**操作（アクション）を選ぶ**。

タスク指向（❌）: 「経費を追加」→ フォーム → 入力 → 保存
オブジェクト指向（✅）: 経費の一覧が見える → 1つ選ぶ → その場で編集できる

### この機能のオブジェクト

```
クライアント
  └── 経費（名前・金額・通貨）  ← 主要オブジェクト
        └── 内訳（オプション）
```

「経費」がユーザーが操作するオブジェクト。画面上に実体として見えていて、直接操作できる。

### OOUI の適用

**1. オブジェクトの一覧性**

経費が「モノ」として一覧に並んでいる。各経費は名前・金額・状態を持つ。一覧を見れば全体像がわかる。

```
サーバ通信費    $45.00/月   USD
Gemini API     自動取得     JPY
```

❌ フォームが主役で、一覧が脇役になってる（現状）
✅ 一覧が主役で、フォームは一覧に対する操作として出現する

**2. オブジェクトから操作へ**

経費オブジェクトをクリック → その経費に対する操作が出る（編集・削除・内訳表示）。

❌ 画面下部に常設の「追加フォーム」がある（タスク指向）
✅ 「+」ボタンで新しい経費オブジェクトが生まれ、その場で編集が始まる

**3. 選択 → アクション パターン**

請求書作成も OOUI で考えると:

- クライアント一覧 → クライアント選択 → その月の請求内容が見える → 「作成」
- 「請求書を作る」タスクではなく、「この月の請求内容」というオブジェクトを確認して確定する

**4. コレクションとシングルトンの区別**

| オブジェクト | コレクション（一覧）       | シングルトン（詳細） |
| ------------ | -------------------------- | -------------------- |
| 経費         | クライアント配下の経費一覧 | 1つの経費の編集画面  |
| 内訳         | 経費配下の内訳一覧         | 1つの内訳の編集      |
| 月次請求     | クライアント × 月の一覧    | 1つの月の請求内容    |

コレクション画面では**一覧性と比較**が重要。
シングルトン画面では**詳細と操作**が重要。

### 現状の UI と OOUI のギャップ

| 現状                           | OOUI 的にあるべき姿                                            |
| ------------------------------ | -------------------------------------------------------------- |
| 追加フォームが常に表示されてる | 一覧が主役。「+」ボタンでインラインまたはダイアログで追加      |
| グループとアイテムの概念が露出 | 「経費」オブジェクトだけ見える。内訳は展開で                   |
| 編集はダイアログで別世界       | できればインラインで直接編集                                   |
| 設定画面と請求書画面が分離     | 請求書画面に経費オブジェクトが埋め込まれ、そこから設定に飛べる |

## この機能への適用

| 原則                               | 請求書作成画面                                 | 経費設定画面                                    |
| ---------------------------------- | ---------------------------------------------- | ----------------------------------------------- |
| 視覚的階層                         | 合計額を最大フォント。明細は中。為替レートは小 | 経費名と金額が主。通貨・税率は副                |
| プログレッシブ・ディスクロージャー | 為替変更は折りたたみ。差額調整は該当時のみ表示 | 内訳は「展開」で表示。追加フォームは「+」で表示 |
| 直接操作                           | 為替レートはクリックで編集                     | 金額はクリックで編集                            |
| フィードバック                     | 作成完了→同画面で結果表示                      | 追加→即座に一覧に反映                           |
| 一貫性                             | 金額は右揃え tabular-nums                      | 同左                                            |
| エラー防止                         | 自動計算値は編集不可                           | 通貨不一致は選択肢で防止                        |
| 空状態                             | 「経費を追加 →」ガイダンス                     | 初回セットアップガイド                          |
| 情報密度                           | 明細テーブル形式                               | 一覧はコンパクト、編集時に展開                  |
| 導線                               | 「経費を編集 →」リンク                         | 「← 請求書作成に戻る」リンク                    |
