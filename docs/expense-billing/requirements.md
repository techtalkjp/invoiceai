# 要件定義: 月次経費請求機能

## 概要

クライアントごとに経費項目（固定費・従量課金）を設定し、為替換算（必要な場合）して月次請求書に自動反映する。

## 現状の課題

特定クライアント向け請求で以下を毎月手作業で算出・入力している:

| 明細                                                  | 種別    | 現状の作業                      |
| ----------------------------------------------------- | ------- | ------------------------------- |
| サーバ通信費 (Vercel $20 + Supabase $15 + Resend $10) | 固定USD | 合計$45を当月末レートで手動換算 |
| Gemini API利用料                                      | 変動USD | GCPコンソールで確認→手動換算    |

## 機能要件

### F1: 経費項目の管理

- クライアントごとに経費項目を登録・編集・削除できる
- 経費項目は2種別:
  - **固定（fixed）**: 毎月固定の外貨金額（例: Vercel $20/月）
  - **従量（metered）**: 外部APIから当月利用額を自動取得（例: GCP Billing）
- 通貨は USD または JPY（GCP 日本アカウントは JPY で返る。JPY の場合は為替換算不要）
- 項目ごとに有効/無効を切り替えられる
- 項目に有効期間（開始月・終了月）を設定できる
  - 「4月から追加」「12月で終了」等のケースに対応
  - 日割りはしない（月単位の on/off）

### F2: 経費グループ

- 複数の経費項目を1つのグループにまとめられる
- グループ単位で請求書の1明細行になる
- グループが通貨を持ち、所属アイテムはすべて同一通貨
- グループには請求書表示名テンプレートを設定できる
  - 例: `"ホゴスルサーバ通信費 {year}年{month}月 (月{amount_foreign}ドル:ドル円{rate}円換算)"`
- グループに属さない項目は単独で1明細行になる
- グループに税率を設定できる（10=課税, 8=軽減, 0=非課税。freee API の `tax_rate` にそのまま渡す）
  - 税率が異なる経費を混在させたい場合はグループを分ける
  - グループ所属アイテムは税率を持たない（グループから継承）
  - 単独項目は自身の税率を持つ

### F3: 為替レート取得

- 日銀 時系列統計データ API から USD/JPY 仲値（TTM）を取得
- 当月末営業日のレートを使用
- 年月 + 通貨ペア単位で DB にキャッシュ
- 手動上書き可能（理由の記録が必須）
- 手動設定したレートは API 再取得で上書きされない（保護される）
- フォールバック: 日銀 API 障害時は手動入力（自動フォールバックはしない — ソースの異なるレートを暗黙で使うのは危険）

### F4: 従量課金の取得

- 対応プロバイダ: Google Cloud（BigQuery Billing Export）
  - Billing Account + プロジェクトID + サービス名でフィルタ
  - `usage_start_time` ベースで利用月を判定（GCP 請求月ではなく実際の利用月）
  - サービスアカウントで認証
  - 月間コストを取得（クレジット差引後の実質コスト）。通貨は Billing Account の設定に依存（日本アカウントは JPY）
- プロバイダはプラグイン的に追加できる設計にする
- 確定タイミング: 翌月5日頃（月末締め後、全データ確定まで数日かかる）
- GCP 集計のフィルタ条件:
  - プロジェクト ID: 必須
  - サービス名: オプション（指定すれば特定サービスのみ集計。未指定ならプロジェクト全体）
  - 共有プロジェクトの配賦は対象外（1プロジェクト = 1クライアント前提）

### F5: 請求書への反映

- 請求書作成時に経費明細を自動生成
- 各経費行の金額算出:
  - 通貨が JPY の場合: 金額をそのまま使用（為替換算なし）
  - 通貨が JPY 以外の場合: 外貨金額 × 為替レート → 端数処理 → 円額
- 端数処理方式はクライアントごとに設定可能（デフォルト: 四捨五入）
- freee 請求書 API に追加の明細行として送信
  - 各経費行の `tax_rate`（10/8/0）を freee API の line item にそのまま渡す
  - 既存の稼働明細と同じ請求書に複数行で送信（現状は1行だが複数行対応が必要）
- **凍結**: 請求書に採用した値（外貨金額・為替レート・円額）は `invoice_line` に保存される。請求書確定後に為替レートや経費実績が変わっても、確定済み請求書の値は変わらない
- **暫定値の運用**: 従量課金は翌月5日頃まで未確定の可能性がある。請求書作成はブロックしない。画面に暫定値である旨を表示する
- **差額調整（metered items のみ）**: 従量課金は API の返却値が確定後に変わりうるため、請求書凍結後に expense_record が再取得で更新された場合、次月の請求書作成画面で差額を表示する。差額の円換算には元請求月の為替レートを使用（為替変動を混ぜない。JPY の場合はそのまま）。調整を反映したら expense_record に調整先の請求書IDと調整時点の金額を記録し、以後は調整済み金額との差分で判定する（二重調整防止・再調整対応）。fixed items は金額が確定的なので差額検出の対象外

### F6: 月次経費実績の記録

- 経費の取得結果（外貨金額・取得日時）を DB に保存
- これは「原データ」であり、請求書の確定値とは独立
- 再取得で更新される（冪等）
- 過去の取得結果を確認・参照できる

## 非機能要件

- **監査性**: 為替レートのソース・取得日時・手動上書き理由を記録する
- **冪等性**: 同月の経費取得を複数回実行しても結果が一貫する
- **凍結性**: 請求書確定後の値は不変。原データの変更が確定済み請求に波及しない

## 技術決定事項

- **通貨**: USD および JPY に対応。固定費は USD、GCP 従量課金は JPY（日本アカウント）。JPY の場合は為替換算をスキップ
- **為替レート**: 日銀 時系列統計データ API（REST、認証不要）。系列コード `FXERD05`（USD/JPY 仲値 TTM 日次）
- **GCP コスト取得**: BigQuery Billing Export の Standard usage cost データ
- **GCP 集計基準**: `usage_start_time` ベース（利用月基準）
- **GCP 認証**: サービスアカウント + `bigquery.dataViewer` / `bigquery.jobUser` ロール
- **金額の精度**: 外貨金額・レートは TEXT（decimal文字列）、円額は INTEGER
- **フォールバック**: 自動フォールバックなし。障害時は手動入力
