# UX フロー: 月次経費請求

## 設計原則

1. **ユーザーは「経費」を考える。** 「グループ」「アイテム」「プロバイダ」はシステムの都合。UI に出さない
2. **毎月の請求書作成が最もスムーズであるべき。** 設定が完了していれば、1画面・1クリック
3. **シンプルなケースはシンプルに。** 「月$45のサーバ代」を登録するのに5ステップも要らない
4. **複雑さは必要な人だけに見せる。** 内訳分解や従量課金設定は、求められたときだけ表示

## 3つの動線

### 動線 1: 毎月の請求書作成（メインフロー）

**目標: 1画面で全情報を確認し、1クリックで完了**

```
請求書作成画面を開く → クライアント・月を選ぶ
  ↓
┌─────────────────────────────────────────┐
│ 稼働明細                                │
│   開発・保守 3月分  16h × ¥20,000       │
│                         = ¥320,000      │
│                                         │
│ 経費                  [経費を編集 →]     │
│   USD/JPY 149.53 (日銀TTM 3/31) [変更]  │
│                                         │
│   サーバ通信費     $45.00 → ¥6,729      │
│   Gemini API利用料        ¥9            │
│                                         │
│ 合計  ¥326,738（税抜）                   │
│                                         │
│ [請求書を作成]                           │
└─────────────────────────────────────────┘
```

- 経費が未設定なら「経費を追加 →」リンクを表示
- 為替レートは自動取得済み。変更は [変更] をクリックしたときだけ
- 暫定値がある場合はその行に小さく表示
- 差額調整がある場合は経費セクション内に自動追加

### 動線 2: 経費の設定・変更（たまにやる）

**入口:**

- 請求書作成画面の「経費を編集 →」リンク（最も自然な入口）
- クライアント詳細の「経費」タブ

**目標: 「何にいくら払ってるか」を直感的に設定**

#### 空状態（初回）

```
┌─────────────────────────────────────────┐
│ このクライアントの経費はまだ登録されて    │
│ いません。                               │
│                                         │
│ 毎月の固定費やクラウドサービスの利用料を  │
│ 登録すると、請求書に自動で反映されます。  │
│                                         │
│ [固定費を追加]  [利用料を自動取得]        │
└─────────────────────────────────────────┘
```

- **「固定費を追加」**: 名前と金額を入力するだけ。最短2フィールド
- **「利用料を自動取得」**: GCP等の従量課金。接続設定を含むガイド付きフロー

#### 固定費の追加（シンプルケース）

```
経費を追加
  名前:   [サーバ通信費        ]
  金額:   [45.00              ] [USD ▼]
  税率:   [10% ▼]

  [保存]
```

これだけ。裏側では expense_group + expense_item が1つずつ作られる。
請求書には「サーバ通信費 2026年3月 (月45ドル:ドル円149円換算)」と自動で載る。

#### 内訳の追加（必要な人だけ）

保存後、経費一覧で:

```
┌─ サーバ通信費 ──── $45.00/月 ──────────┐
│                            [編集] [削除] │
│                                         │
│ 内訳はありません。                       │
│ [内訳を追加] ← 押すと分解できる          │
└─────────────────────────────────────────┘
```

「内訳を追加」を押すと:

```
┌─ サーバ通信費 ──── $45.00/月 ──────────┐
│                            [編集] [削除] │
│                                         │
│ 内訳:                                   │
│   Vercel      $20.00  [×]               │
│   Supabase    $15.00  [×]               │
│   Resend      $10.00  [×]               │
│                                         │
│   [+] ───── [      ] [     ] [追加]     │
└─────────────────────────────────────────┘
```

内訳はオプション。登録しなければ合計額がそのまま使われる。
登録すれば、内訳の合計が経費の金額になる（合計額フィールドは非表示に）。

#### 従量課金の追加（ガイド付き）

「利用料を自動取得」を押すと:

```
ステップ 1/3: サービスを選択
  [Google Cloud (Gemini API等)]
  [Anthropic (Claude API)]     ← 将来
  [OpenAI (ChatGPT API)]       ← 将来

ステップ 2/3: 接続設定
  サービスアカウントの JSON キーが必要です。
  [作成手順を見る]

  [ファイルを選択] または [ペースト]
  ✓ 認証成功

ステップ 3/3: 取得対象を設定
  名前:         [Gemini API利用料    ]
  プロジェクト:  [dailove-search      ]
  サービス:     [Cloud AI API   ▼] (任意)
  税率:         [10% ▼]

  [保存]
```

GCP の認証情報がすでに登録済みなら、ステップ2はスキップ。

#### 設定済みの状態

```
┌─ 経費 ──────────── [← 請求書作成に戻る] ┐
│                                         │
│ サーバ通信費          $45.00/月  [編集]  │
│   Vercel $20 / Supabase $15 / Resend $10│
│                                         │
│ Gemini API利用料      自動取得   [編集]  │
│   GCP · dailove-search                  │
│                                         │
│ [+ 経費を追加]                           │
│                                         │
└─────────────────────────────────────────┘
```

シンプル。名前・金額・状態が一目でわかる。

### 動線 3: 過去の経費を確認する

**入口:** クライアント詳細の「経費実績」タブ

**目的:** 月別の経費実績を振り返る。税理士への報告や、コスト推移の確認に使う。

```
┌─ 2026年3月 ────────────────────────────┐
│ USD/JPY: 149.53                         │
│ サーバ通信費    $45.00 → ¥6,729  請求済 │
│ Gemini API      ¥9              請求済 │
├─ 2026年2月 ────────────────────────────┤
│ USD/JPY: 155.50                         │
│ サーバ通信費    $45.00 → ¥6,998  請求済 │
│ Gemini API      ¥15             請求済 │
└─────────────────────────────────────────┘
```

## データモデルとUIの対応

| ユーザーが見るもの            | UI での呼び方                  | 内部データモデル                                         |
| ----------------------------- | ------------------------------ | -------------------------------------------------------- |
| 1つの経費（例: サーバ通信費） | 経費                           | expense_group（内訳なしなら expense_item も1つ自動作成） |
| 内訳（例: Vercel $20）        | 内訳                           | expense_item（group 配下）                               |
| 自動取得の経費                | 自動取得の経費                 | expense_item（type=metered）                             |
| GCP の接続設定                | （従量課金追加フロー内に統合） | provider_credential                                      |

**重要: 「グループ」「アイテム」という用語は UI に一切出さない。**

「経費を追加」すると、裏側では:

1. expense_group が1つ作られる（名前、通貨、税率）
2. expense_item が1つ作られる（type=fixed, monthly_amount=入力値）
3. 内訳を追加すると、既存の expense_item を分割する（group の下に複数 item）

## 請求書作成画面と経費設定の導線

```
請求書作成画面 ──「経費を編集 →」──→ 経費設定画面
                                      │
経費設定画面 ──「← 請求書作成に戻る」──→ 請求書作成画面
                                      │
クライアント詳細 ── 経費タブ ──→ 経費設定画面（同じ画面）
```

「請求書作成に戻る」リンクは、請求書作成画面から来た場合のみ表示。
クライアント詳細から来た場合は通常のパンくずリスト。
